  3. O que foi implementado

  Workflows n8n (2)

  - webhook-http-validator.json — Trigger via POST webhook; valida URL recebida no body; retorna JSON com status,
  error_type e timestamp
  - http-status-validator.json — Trigger manual; mesmo fluxo com Merge node para consolidar resultados

  Ambos tratam: 200 (sucesso), 4xx (client errors com mapeamento de tipo), outros erros

  Infraestrutura Terraform

  - AWS: VPC + subnets + ECS Fargate + ALB + EFS + CloudWatch + IAM roles
  - Azure: Resource Group + Storage Account + App Service + Application Insights

  Scripts de Teste

  - test-requests.sh / .bat — 5 cenários (200, 404, 401, 403, 500)
  - test-all.sh — vazio (placeholder)

  Documentação (5 arquivos Markdown)

  README, QUICKSTART, CONTRIBUTING, TESTING, WORKFLOWS — nível excepcional
